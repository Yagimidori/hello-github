# note記事ドラフト生成

引数: $ARGUMENTS（テーマや要件。省略時はトレンドから自動選定）

## 手順

### 1. サブエージェントでトピック調査
`note-researcher` サブエージェントを使い、以下を調査する:
- 今週のnote.comトレンドトピック（WebSearchで調査）
- 指定テーマがある場合はその関連情報収集
- 競合記事の構成・トーン分析

調査結果をもとに**記事テーマ3案**を提示し、ユーザーに選択させる（引数でテーマ指定済みの場合はスキップ）。

### 2. 記事構成の設計
`note-writer` サブエージェントに以下を指示:
- 対象読者の明確化
- タイトル案3パターン（30文字以内）
- 見出し構成（H2: 3〜5個、H3: 各H2に1〜3個）
- 推定読了時間（目標: 5〜8分 = 約2000〜3000文字）

### 3. 本文執筆
`note-writer` サブエージェントが本文を執筆:
- 書き出しは読者の「悩み・疑問」から始める
- 各セクションに具体例・データ・体験談を含める
- まとめには行動を促すCTAを入れる
- タグ候補5個を末尾に記載

### 4. ドラフト保存
以下のフォーマットで `drafts/YYYY-MM-DD-タイトル.md` に保存:

```markdown
---
title: "記事タイトル"
tags: ["タグ1", "タグ2", "タグ3"]
created: YYYY-MM-DD
status: draft
estimated_read: X分
---

# 記事タイトル

本文...
```

### 5. 完了報告
- ファイルパスを表示
- 文字数・推定読了時間を表示
- `/note-post` で投稿できることを案内
