# Claude Codeでnote記事を自動生成してみた

「毎週記事を書きたいけど、ネタ切れが怖い」「書くのに時間がかかりすぎる」——そんな悩みを抱えているnoteクリエイターは多いはず。

実は私もそのひとりでした。週1投稿を目標にしていたのに、いつの間にか月1すら怠くなっていた。でも**Claude Code**と出会って、その状況が一変しました。

## Claude Codeって何？

Claude CodeはAnthropicが開発したAIコーディングツールですが、コードを書くだけではありません。トレンド調査、記事執筆、投稿の自動化まで——**あらゆる「繰り返し作業」を肩代わりしてくれます**。

具体的にできることを挙げると:
- 今週のトレンドトピックをWebから自動収集
- 2000〜3000文字の記事本文を数分で生成
- 毎週月曜日に自動でドラフトを作成
- Playwrightでnote.comに自動投稿

## 実際の運用フロー

私が構築した自動化パイプラインはこうです。

### 1. 週次スケジュールタスクでドラフト生成

毎週月曜の朝9時、Claude Codeがバックグラウンドで動き始めます。

```
/schedule 毎週月曜9時にnote記事ドラフトを自動生成
```

このコマンド1つで、Anthropicのクラウド上でClaudeが動き続けます。あなたのPCが起動していなくてもOK。

### 2. トレンド調査 → テーマ選定

`note-researcher`エージェントがX(Twitter)やGoogleトレンドを横断して今週の旬なネタを収集。競合記事を分析して「まだ誰も書いていない切り口」を見つけます。

### 3. 高品質な本文を自動執筆

`note-writer`エージェントが選定したテーマで本文を執筆。読者の悩みから始まり、具体例・データ・CTAまで含んだ「バズる構造」で仕上げてくれます。

### 4. 確認して投稿

生成されたドラフトはGitHubのPRとして届きます。内容を確認してOKなら:

```
/note-post
```

これだけ。Playwrightがブラウザを自動操作してnote.comに投稿してくれます。

## 実際に使ってみた感想

**圧倒的な時間短縮**が実感できます。以前は1記事に3〜4時間かかっていましたが、今は確認・微修正だけなので30分以内に収まります。

もちろん「AIが書いた記事はバレる」という心配もありました。でも`note-writer`エージェントには具体的な文体指示を詳しく設定しているので、読んでいても自然です。むしろ「よく構成が整っているな」と言われることが増えました。

## まず試してほしいこと

今日からできることを3つ挙げます:

1. **Claude Codeをインストールする**（`curl -fsSL https://claude.ai/install.sh | bash`）
2. **CLAUDE.mdにあなたの執筆スタイルを書き込む**（ペルソナ、禁止表現、好きな構成など）
3. **`/note-draft` で最初のドラフトを生成する**

note運用に悩んでいる方、ぜひ試してみてください。もしこの記事が参考になったら「スキ」をいただけると嬉しいです！
